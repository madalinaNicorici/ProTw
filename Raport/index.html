Descriere

Fun@Web este un joc Web multi-player care are drept scop ilustrarea folosirii in practica si invatarea de catre jucatori a celor mai importante concepte predate la disciplina "Tehnologii Web": protocolul HTTP, cookie-uri, sesiuni Web, mecanismul de functionare a unui server de aplicatii Web, dezvoltarea si invocarea de servicii Web pe baza REST, transferul asincron de date prin Ajax, realizarea de mash-up-uri, aspecte esentiale privind securitatea Web si altele. Pe baza API-ului REST creat, jocul poate fi jucat si in grup via o platforma sociala precum Facebook, inclusiv pe telefoane mobile.

Tehnologii folosite

HTML5

HyperText Markup Language (HTML) este un limbaj de marcare utilizat pentru crearea paginilor web ce pot fi afișate într-un browser. HTML furnizează mijloacele prin care conținutul unui document poate fi adnotat cu diverse tipuri de metadate și indicații de redare. HTML5 este ultimul standard HTML si vine cu un set bogat de noi posibilitati de programare in navigatorul web. Cu ajutorul acestui limbaj se creaza interfata grafica, partea vizuala a aplicatiei.

CSS3

CSS (Cascading Style Sheets) este un standard pentru formatarea elementelor documentului HTML. Stilurile se pot atașa elementelor HTML prin intermediul unor fișiere externe. CSS3 reprezintă un upgrade ce aduce câteva atribute noi și ajută la dezvoltarea noilor concepte in webdesign. Unele dintre cele mai importante segmente (module) noi adăugate acestui standard pentru formatarea elementelor HTML aduc un plus considerabil in dezvoltarea activități webdesign.

PHP

Pentru partea de back-end a aplicatiei se va recurge la un limbaj de programare precum PHP. PHP este simplu de utilizat, fiind un limbaj de programare structurat, ca și C-ul, Perl-ul sau începând de la versiunea 5 chiar Java, sintaxa limbajului fiind o combinație a celor trei. Probabil una din cele mai importante facilități ale limbajului este conlucrarea cu majoritatea bazelor de date relaționale, de la MySQL și până la Oracle, trecând prin MS Sql Server, PostgreSQL, sau DB2. PHP poate rula pe majoritatea sistemelor de operare, de la UNIX, Windows, sau Mac OS X și poate interacționa cu majoritatea serverelor web.

MySQL Database

MySQL este un sistem de gestiune a bazelor de date relaționale, produs de compania suedeza MySQL AB și distribuit sub Licența Publică Generală GNU. Este cel mai popular SGBD open-source la ora actuală[6], fiind o componentă cheie a stivei LAMP (Linux, Apache, MySQL, PHP). Deși este folosit foarte des împreună cu limbajul de programare PHP, cu MySQL se pot construi aplicații în orice limbaj major. Există multe scheme API disponibile pentru MySQL ce permit scrierea aplicațiilor în numeroase limbaje de programare pentru accesarea bazelor de date MySQL

Ajax

Ajax (sau AJAX), prescurtare pentru Asynchronous JavaScript and XML, este o tehnică de programare pentru crearea de aplicații web interactive. Intenția este să facă paginile web să devină mai rapide și deci mai acceptate, prin schimbul în fundal al unor cantități mici de date cu serverul, astfel încât să nu fie nevoie ca pagina să fie reîncărcată la fiecare acțiune a utilizatorului. Aceasta are ca scop creșterea interactivității, vitezei și ușurinței în utilizare a aplicațiilor web. A fost folosit pentru deschiderea paginilor de login si de sign-up si la quiz-ul de la singleplayer si de la multiplayer.

BOOTSTRAP

BOOTSTRAP este un framework de dezvoltare a aplicatiilor web in mod responsive. Acesta faciliteaza scalarea paginilor web si a aplcatiilor, independent de dispozitivele pe care ruleaza si ofera un set predefinit de stiluri definite pentru toate elementele HTML.

REST

REST API este un stil arhitectural de dezvoltare software ce consta intr-un set de constrangeri aplicate asupra componentelor, conectorilor si a elementelor. Aceste constrangeri sunt legate de: arhitectura Client/Server (serverul ofera clientului serviciul cerut), server stateless (orice cerere este noua pentru server), atat clientul cat si serverul dispun de memorie Cache, clientul nu poate sesiza daca este conectat direct la server sau nu, serverul poate customiza temporar functionalitatea clientului prin transfer de cod executabil si separarea resurselor de reprezentare.

JavaScript

JavaScript (JS) este un limbaj de programare orientat obiect bazat pe conceptul prototipurilor.[5] Este folosit mai ales pentru introducerea unor funcționalități în paginile web, codul Javascript din aceste pagini fiind rulat de către browser. Limbajul este binecunoscut pentru folosirea sa în construirea siturilor web, dar este folosit și pentru acesul la obiecte încastrate (embedded objects) în alte aplicații. A fost dezvoltat inițial de către Brendan Eich de la Netscape Communications Corporation sub numele de Mocha, apoi LiveScript, și denumit în final JavaScript.

Use Case-uri:

I. Log In

  1. Utilizatorul acceseaza situl.
  2. Pentru a avea acces la jocuri, utilizatorul trebuie sa se autentifice. Daca nu este autentificat, ii va aparea o pagina pe care va avea posibilitatea sa faca ”Login”, ”Sign Up”.
  3. Utilizatorul da click pe butonul ”Log In” de pe pagina principala si va fi directionat pe pagina de login.
  4. Aici va trebui sa introduca un nume de utilizator existent si parola corespunzatoare, apoi sa apese butonul ”Sign In”. Daca nu va reusi sa se autentifice, si de pe aceasta pagina are posibilitatea sa se inregistreze.
  5. Dupa ce a efectuat acest pas, ii va aparea pagina de pe care va putea sa opteze sa joace jocul in versiunea single player si multi player.

II. Sign Up

  1. Utilizatorul acceseaza situl.
  2. Pentru a avea acces la jocuri, utilizatorul trebuie sa se autentifice. Daca nu este autentificat, ii va aparea o pagina pe care va avea posibilitatea sa faca ”Login”, ”Sign Up”.
  3. Utilizatorul opteaza pentru inregistrare, apasand butonul ”Sign Up”.
  4. Utilizatorului ii va aparea un formular unde trebuie sa introduca nume,prenume, o adresa de email valida, un nume de utilizator, si o parola.
  5. Dupa ce a efectuat acest pas, ii va aparea pagina de pe care va putea sa opteze sa joace jocul in versiunea single player si multi player.

III. Joc single player

Utilizatorul are posibilitatea de a juca singur jocul, rezolvand teste pentru a acumula punctaj si pentru a se familiariza cu tipul intrebarilor.

  1. Utilizatorul acceseaza situl.
  2. Pentru a avea acces la jocuri, utilizatorul trebuie sa se autentifice. Daca nu este autentificat, ii va aparea o pagina pe care va avea posibilitatea sa faca ”Login”, ”Sign Up”.
  3. . Dupa autentificare(sau, dupa caz, daca este deja autentificat), utilizatorul poate opta sa joace, dand click pe optiunea ”Play Game”. In meniul aplicatiei ii va aparea si optiunea ”Sign out”.
  4. Din pagina care i se afiseaza, utilizatorul selecteaza butonul ”Single Player”.
  5. De aici ii va aparea butonul ”Start Game”, pe care il va apasa cand va fi pregatit sa inceapa jocul.
  6. . Utilizatorului ii va aparea o lista cu zece intrebari care vor avea patru variante de raspuns dintre care va putea alege doar una pentru fiecare intrebare. Dupa ce va trece de ultima intrebare, ii va fi calculat punctajul, care se va adauga la punctajul lui total.
  7. Pe aceeasi pagina vor fi afisate butoanele ”Play Again” si ”Back”.

IV. Joc multi player

  1. Utilizatorul acceseaza situl.
  2. Pentru a avea acces la jocuri, utilizatorul trebuie sa se autentifice. Daca nu este autentificat, ii va aparea o pagina pe care va avea posibilitatea sa faca ”Login”, ”Sign Up”.
  3. Dupa autentificare(sau, dupa caz, daca este deja autentificat), utilizatorul poate opta sa joace, dand click pe optiunea ”Play Game”. In meniul aplicatiei ii va aparea si optiunea ”Sign out”.
  4. Din pagina care i se afiseaza, utilizatorul selecteaza butonul ”Multi Player”.
  5. . Utilizatorul este directionat la pagina pe care se afla grupurile care sunt formate, dar nu joaca inca.
    1. Utilizatorul are posibilitatea de a-si crea propriul grup. Daca doreste, ii poate asocia o parola, pentru a permite doar jucatorilor cunoscuti sa intre in joc.
    2. Utilizatorul poate sa se alature unui grup deja creat, daca acesta nu este parolat sau daca utilizatorul cunoaste parola.
  6. Dupa ce intr-un grup sunt macar doi jucatori(maxim 4), acestia au posibilitatea de a apasa butonul ”Start Game”.
  7. Utilizatorii vor primi cate o intrebare cu patru variante de raspuns si un singur raspuns corect si vor avea 20 desecunde pentru a raspunde la fiecare intrebare.
  8. . . Daca un utilizator a terminat de raspuns, apasa ”Next Question”. Daca nu au terminat de raspuns si ceilalti, el va trebui sa ii astepte. Daca un utilizator nu apasa ”Next Question”, la expirarea celor 20 de secunde se va trece automat la intrebarea urmatoare.
  9. . Dupa 10 intrebari, jocul va lua sfarsit. Punctajele acumulate de fiecare jucator vor fi adunate la punctajul curent al acestora.
  10. Fiecarui jucator i se va afisa un clasament cu punctajele si ii vor aparea optiunile ”Play Another Game”, ”Play Single Player”, ”Learn”.

Daca un jucator va parasi jocul inainte de incheierea acestuia, el nu va influenta cu nimic experinta celorlalti jucatori, ci va primi 0 puncte pentru fiecare intrebare.

Etapele intermediare

1. Crearea fisierelor HTML si CSS pentru o viziune preliminara a aspectului sitului. Sunt implemetate in special paginile care nu sunt prea importante in logica aplicatiei, dar vor fi esentiale pentru a familiariza utilizatorul cu mediul acesteia.

2. Modelarea unei ierarhii de clase cu ajutorul unor diagrame de clase.

3. Implemetarea, la toate nivelurile aplicatiei, a functionalitatilor de baza: autentificare, inregistrare, conectarea cu Facebook. La nivelul Model se vor crea baze de date cu informatii despre utilizatori. La nivelul View, se vor crea formulare, iar la nivelul Controller se va face legatura dintre datele primite prin formulare si datele din baza de date. Pentru conectarea cu Facebook se va folosi un API extern, cum ar fi RestFB.

4. Crearea modului single player. La nivelul Model, se vor crea tabele cu intreabri pentru chestionare. La nivelul View se vor afisa intrebarile sub forma unui chestionar, iar raspunsurile vor putea fi selectate cu ajutorul butoanelor radio. Controller-ul se va ocupa ca intrebarile sa fie preluate din baza de date si afisate pe sait, iar raspunsurile utilizatorului vor fi verificte daca sunt in conformitate cu raspusurile corecte din baza de date. Daca un raspuns nu este corect, la sfarsitul chestionarului Controller-ul va aduce din baza de date in View explicatiile pentru motivarea raspunsului corect. Punctajul obtinut de utilizator se va afisa in View si va produce modificari in Model, prin intermediul Controller-ului.

5. Crearea modului multi player. La nivelul Model, se vor utiliza tabelel cu intrebari deja create. La niveulul View, intrebarile se vor afisa individual, impreuna cu un temporizator. Nivelul Controller, in afara functionalitatilor asemanatoare cu cele de la single player, va asigura un mod concurent de joc, avand grija ca activitatea diferitelor grupuri sa nu se suprapuna.

Arhitectura

Modelul arhitectural folosit este MVC(Model View Controller) implementat cu ajutorul framework-ului Codeigniter. Acest pattern imparte aplicatia in doua nivele: nivelul prezentare si nivelul modelare. Nivelul prezentare este de asemenea impartit in nivelele view si controller.

1. Model
Modelul este o reprezentare a informatiei cu care opereaza aplicatia. Pentru modelarea datelor se foloseste o baza de date MySQL.

2. View
View-ul afiseaza datele din model intr-o forma potrivita pentru interactiuni. View-ul va fi realizat intr-un format web, cu ajutorul unor fisiere HTML,CSS si JavaScript.

3. Controller
Controller-ul raspunde la evenimentele aparute in urma actiunilor utilizatorului si le proceseaza, putand aduce modificari in model. In aplicatia noastra, acesta va fi asculta pentru cereri HTTP, va extrage datele importante din eveniment si le va procesa. Controller-ul va fi implementat in PHP.

Motivatie

Situl ofera posibilitatea invatarii notiunilor de tehnologii web prin metode interactive. Un utilizator complet nefamiliar cu acestea, are posibilitatea sa invete, accesand optiunea ”Learning”, un utilizator ce are cunostinte medii va putea sa joace jocul single player care ii va afisa raspunsurile corecte si explicatii dupa terminarea testului, iar un utilizator care are incredere in cunostintele lui va putea sa joace jocul multi player impotriva altor utilizatori, pentru a-si testa capacitatile.

Deoarece acest sit are ca scop invatarea, designul este simplu, iar posibilitatile de distragere a utilizatorului de la continutul sitului sun limitate. Informatia este prezentata intr-un mod elegant, evitandu-se suprapopularea paginilor.
Situl ofera flexibilitate. Meniul este accesibil pe fiecare pagina acestuia(eventual cu cateva optiuni specifice adaugate in diferite zone) si navigarea prin sit este ghidata de butoane si mesaje explicative.
